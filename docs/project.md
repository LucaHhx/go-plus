## 产品核心价值

- 提供丰富的在线赌场游戏体验 (Slots, Live, Crash, Table Game, Fishing, Lotto)
- 集成多家游戏供应商 (JILI, Spribe, JDB, Evolution, Playtech 等 18+家)
- 即时充值提现 (UPI, Paytm, GPay, Amazon Pay)
- 完善的用户激励体系 (VIP, 促销, 奖池, 抽奖, 任务, 推广佣金)
- 移动优先的深色主题 UI 设计

## 目标用户

- 印度市场的在线赌场玩家
- 体育博彩爱好者
- 通过推广系统引入的推荐用户

## 分期规划

| 期数 | 主题 | 目标 |
|------|------|------|
| 第一期 | 全部页面 + mock 数据 | 前端 1:1 还原所有设计稿页面和组件，后端核心链路全功能，其余 mock |
| 第二期 | 真实业务接口 | 替换 mock → 真实后端，上线促销/VIP/奖池/实时投注业务逻辑 |
| 第三期 | 裂变与生态 | 替换占位 → 完整功能，上线推广/抽奖/任务/幸运转盘 |

### 第一期交付策略

**核心原则: 第一期交付设计图纸上的所有页面和所有组件，前端 1:1 还原，后端未就绪的功能使用 mock 数据。**

- 前端: 完整还原 merge.html 设计稿中的每一个页面、每一个组件的视觉效果和交互
- 后端: 核心交易链路 (注册/充值/游戏/提现) 为全功能实现，其余提供 mock API
- 占位: 底部 Tab 中的 GET 1700 / Raffle / Quest 显示图标，点击弹出 "Coming Soon"

后端 mock 组件明细见 `docs/tasks.md` 中的 mock 组件明细表。

## 核心设计要求

**1:1 完全还原 UI 设计图纸 (merge.html) 的页面设计和布局。** 所有面向用户的前端页面（首页、注册/登录、游戏大厅、钱包、客服入口等）必须在视觉效果、布局结构、交互方式上与设计图纸保持完全一致。具体要求:

- 页面布局结构与设计图纸一致（元素位置、间距、层级关系）
- 配色方案、字体、字号、字重与设计图纸一致
- 图标、Logo、Banner 图片等视觉资源从设计图纸提取或精确还原
- 交互动效（Tab 切换、侧边菜单滑出、Banner 轮播等）与设计图纸一致
- 组件样式（按钮、卡片、输入框、标签等）与设计图纸一致
- 移动端适配效果与设计图纸一致

> 管理后台（admin-panel）不要求与设计图纸一致，采用标准管理后台设计即可。

## 交付优先级

核心交易链路 (注册 > 充值 > 游戏 > 提现) > 浏览体验 (首页/大厅) > 运营支撑 (客服/管理后台)
