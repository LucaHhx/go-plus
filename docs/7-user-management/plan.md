# 用户管理

> 为已登录用户提供个人资料管理、安全设置和登出功能的完整界面

## 目标

1. 用户可以在界面上找到并执行登出操作
2. 用户可以查看和编辑个人资料（昵称、头像等）
3. 用户可以查看账户基本信息（手机号、注册时间等）
4. 用户可以修改登录密码
5. 用户可以绑定或解绑 Google 账号

## 范围

**包含:**

- 登出功能的前端入口（按钮位置、交互流程、状态清除）
- 个人资料页面（查看手机号、注册时间、VIP 等级等账户信息）
- 昵称编辑功能
- 头像上传和更换功能
- 修改密码功能（需验证当前密码）
- Google 账号绑定/解绑功能
- 个人资料页面的视觉效果与平台整体设计风格一致

**不包含 (不属于本需求范围):**

- 用户注册和登录流程（已在 1-user-system 中完成）
- 会话管理后端逻辑（登出 API 已在 1-user-system 中实现）
- VIP 等级体系的业务逻辑（归属 VIP 系统需求，本需求仅展示等级信息）
- 实名认证 / KYC 验证
- 手机号更换（涉及安全风险较高，不在 MVP 范围内）
- 账户注销/删除

## 核心用户场景

- 场景 A: 用户想要退出当前账户。在侧边菜单或个人资料页面找到登出按钮，点击后弹出确认提示，确认后清除登录状态，页面回到未登录的首页状态。
- 场景 B: 用户想要设置个性化资料。进入个人资料页面，可以看到自己的手机号、注册时间等基本信息，点击编辑昵称或更换头像，保存后立即生效。
- 场景 C: 用户发现密码可能泄露，想要修改密码。进入安全设置，输入当前密码和新密码，验证通过后密码更新成功，其他设备的登录状态自动失效。
- 场景 D: 用户之前用手机号注册，现在想关联 Google 账号以便快捷登录。进入安全设置，点击绑定 Google 账号，完成 Google 授权后绑定成功。后续可以用 Google 账号直接登录。
- 场景 E: 用户想要解除 Google 账号关联。进入安全设置，点击解绑 Google 账号，确认后解除关联。

## 背景

当前平台已完成用户注册和登录功能（1-user-system），后端也已实现登出 API 和用户信息查询接口。但前端缺少完整的用户管理界面：用户无法在界面上找到登出入口，也没有查看/编辑个人资料和修改密码的页面。这导致用户无法主动退出账户、无法个性化自己的资料、无法在密码泄露时自行修改密码。

本需求旨在补齐用户管理的前端体验，让用户对自己的账户拥有完整的控制能力。

## 与 1-user-system 的边界划分

| 功能点 | 1-user-system | 7-user-management |
|--------|--------------|-------------------|

| 用户信息查询 API | 已实现 (GET /auth/me) | 页面展示和编辑 |
| 个人资料编辑 | 不涉及 | 负责 |
| 修改密码 | 不涉及 | 负责 |

## 验收清单

### 登出功能

- [ ] 用户可以在侧边菜单或个人资料页面找到登出按钮
- [ ] 点击登出后弹出确认对话框，防止误操作
- [ ] 确认登出后清除本地登录状态，页面切换为未登录状态
- [ ] 登出后自动跳转到首页

### 个人资料

- [ ] 用户可以进入个人资料页面查看手机号、注册时间等账户信息
- [ ] 用户可以编辑和保存昵称
- [ ] 用户可以上传和更换头像（支持从相册选择或拍照）
- [ ] 头像上传支持裁剪（正方形裁剪框）
- [ ] 资料修改后即时生效，页面上其他位置（如顶栏）同步更新显示

### 安全设置

- [ ] 用户可以修改登录密码（需输入当前密码 + 新密码 + 确认新密码）
- [ ] 密码修改成功后提示用户，其他设备登录状态自动失效
- [ ] 用户可以绑定 Google 账号（未绑定状态显示"绑定"按钮）
- [ ] 用户可以解绑 Google 账号（已绑定状态显示"解绑"按钮，需确认）
- [ ] 如果用户仅有 Google 登录方式（无密码），解绑时需提示先设置密码

### 视觉与体验

- [ ] 个人资料页面的整体风格与平台深色主题一致
- [ ] 页面布局移动优先，适配不同屏幕尺寸
- [ ] 表单交互体验流畅（加载状态、成功/失败反馈、输入校验）
