# QA 测试方案 -- 用户管理

> 需求: 7-user-management | 角色: qa

## 测试策略

本需求涉及用户管理的核心功能（个人资料、密码安全、第三方账号绑定），需覆盖前后端集成测试和边界条件。

### 测试范围

| 功能模块 | 测试重点 | 对应 L2 任务 |
|----------|----------|-------------|
| 登出 | 前端交互流程、状态清除、页面跳转 | #1 |
| 个人资料查看 | 账户信息展示正确性 | #2 |
| 昵称编辑 | 保存成功、校验规则、实时更新 | #3 |
| 头像上传 | 文件格式/大小校验、裁剪、上传成功 | #4 |
| 修改密码 | 当前密码校验、强度校验、Token 刷新、其他设备失效 | #5 |
| Google 绑定 | 绑定流程、重复绑定检测 | #6 |
| Google 解绑 | 解绑确认、唯一登录方式保护 | #7 |
| 视觉一致性 | 深色主题、移动端适配、表单交互反馈 | #8 |

### 测试类型

| 类型 | 范围 | 工具 |
|------|------|------|
| API 测试 | 后端 5 个新增端点的功能和错误码验证 | HTTP 客户端手动测试 |
| UI 功能测试 | 页面交互流程、表单校验、状态同步 | 浏览器手动测试 |
| 集成测试 | 前后端联调、数据一致性 | Dev 环境 |
| 边界测试 | 文件大小超限、密码强度不足、唯一登录方式保护 | 手动测试 |

## 测试用例设计

### 1. 登出功能

| 用例 | 步骤 | 预期结果 |
|------|------|----------|
| 侧边菜单登出 | 打开侧边菜单 > 点击 Logout > 确认 | 清除登录状态，跳转首页 |
| ProfilePage 登出 | 进入 /profile > 点击 Logout > 确认 | 同上 |
| 取消登出 | 点击 Logout > 取消 | 保持登录状态不变 |
| 登出后访问受保护页面 | 登出后手动访问 /profile | 重定向到登录页 |

### 2. 个人资料查看

| 用例 | 步骤 | 预期结果 |
|------|------|----------|
| 展示账户信息 | 登录后进入 /profile | 显示手机号(部分遮罩)、注册时间、VIP 等级 |
| 手机号遮罩格式 | 查看手机号显示 | 格式为 +91****7890 |

### 3. 昵称编辑

| 用例 | 步骤 | 预期结果 |
|------|------|----------|
| 正常编辑 | 点击编辑 > 输入新昵称 > 确认 | 昵称更新成功，页面同步显示 |
| 空昵称 | 清空昵称 > 确认 | 提示不能为空 |
| 超长昵称 | 输入 >20 字符 | 提示超出长度限制 (2-20 字符) |
| 特殊字符 | 输入含空格/特殊符号的昵称 | 提示仅允许字母、数字、下划线 |
| 侧边菜单同步 | 修改昵称后打开侧边菜单 | 显示新昵称 |

### 4. 头像上传

| 用例 | 步骤 | 预期结果 |
|------|------|----------|
| 正常上传 JPEG | 选择 JPEG 图片 > 裁剪 > 确认 | 头像更新成功 |
| 正常上传 PNG | 选择 PNG 图片 > 裁剪 > 确认 | 头像更新成功 |
| 不支持的格式 | 选择 GIF/BMP 文件 | 提示格式不支持 |
| 超大文件 | 选择 >2MB 图片 | 提示文件过大 |
| 裁剪功能 | 选择图片后进入裁剪 | 可缩放、拖拽，正方形裁剪框 |
| 取消裁剪 | 选择图片后点击取消 | 头像不变 |

### 5. 修改密码

| 用例 | 步骤 | 预期结果 |
|------|------|----------|
| 正常修改 | 输入正确的当前密码 + 有效新密码 | 密码修改成功，返回新 Token |
| 当前密码错误 | 输入错误的当前密码 | 提示密码错误 (1004) |
| 新密码不一致 | 新密码与确认密码不同 | 提示不一致 (1010) |
| 密码强度不足 | 新密码 <6 位或纯字母/纯数字 | 提示强度不足 (1011) |
| 其他设备失效 | 修改密码后用旧 Token 请求 | 返回 401 (Token 失效) |
| Google-only 用户首次设密码 | 无当前密码字段 > 设置新密码 | 密码设置成功 |

### 6. Google 账号绑定

| 用例 | 步骤 | 预期结果 |
|------|------|----------|
| 正常绑定 | 点击绑定 > 完成 Google 授权 | 显示已绑定的邮箱 |
| 已被其他用户绑定 | 使用已关联的 Google 账号 | 提示已被占用 (1012) |
| 当前用户已绑定 | 已绑定状态再次点击绑定 | 提示已绑定 (1013) |

### 7. Google 账号解绑

| 用例 | 步骤 | 预期结果 |
|------|------|----------|
| 有密码用户解绑 | 点击解绑 > 确认 | 解绑成功，显示绑定按钮 |
| 唯一登录方式保护 | Google-only 用户 (无密码) 点击解绑 | 提示先设置密码 (1015) |
| 取消解绑 | 点击解绑 > 取消 | 保持绑定状态 |

### 8. 视觉与体验

| 用例 | 检查点 | 预期结果 |
|------|--------|----------|
| 深色主题一致性 | 所有页面背景、卡片、输入框 | 与平台整体风格一致 |
| 移动端适配 | 不同屏幕宽度 | 布局正常，无溢出 |
| 加载状态 | API 请求期间 | 显示 loading 状态，按钮禁用 |
| 成功反馈 | 操作成功后 | 显示成功提示 |
| 错误反馈 | API 返回错误 | 显示对应错误信息 |

## 前置条件

- 已有注册和登录功能正常工作 (1-user-system)
- 后端新增 API 端点已部署
- 测试账户: 纯手机号用户、Google-only 用户、手机号+Google 混合用户

## 关键决策

1. **手动测试为主:** MVP 阶段以手动功能测试为主，重点验证核心流程和边界条件。
2. **密码安全重点测试:** password_version 机制的多设备失效场景需重点验证。
3. **Google-only 用户场景:** 需要专门准备 Google-only 测试账户来验证无密码场景。
