# 手机端适配优化

> 优化客户端 (frontend) 的手机端浏览体验，确保导航栏固定、内容区独立滚动、各页面适配主流手机屏幕。

## 目标

1. 底部导航栏 (BottomTabBar) 固定在视口底部，不参与页面滚动
2. 顶部导航栏 (TopBar) 固定在视口顶部，不参与页面滚动
3. 页面内容区域独立滚动，不影响导航栏位置
4. 所有页面在主流手机分辨率下显示正常，无水平溢出或内容截断
5. 触摸交互体验符合移动端操作习惯

## 范围

**包含:**

- 底部导航栏 (BottomTabBar) 固定定位优化
- 顶部导航栏 (TopBar) 固定定位优化
- 页面内容区域滚动机制优化（内容区独立滚动容器）
- 各主要页面手机端布局适配检查与修复
  - 首页 (Home)
  - 游戏大厅 (Game Lobby)
  - 钱包页 (Wallet: 充值/提现/交易记录)
  - 个人中心 (Profile)
  - 侧边菜单 (SideDrawer)
- 表单组件手机端适配（输入框、按钮、金额选择等）
- 长列表/长页面滚动体验优化
- 触摸操作区域大小优化（按钮、可点击元素的最小触控区域）

**不包含:**

- admin 后台管理系统的手机端适配（admin 以桌面端使用为主）
- 新功能开发（本需求仅优化现有页面的手机端体验）
- 横屏模式适配
- 平板电脑专用布局

## 核心用户场景

- **场景 A**: 用户在手机上浏览首页，页面内容较长需要滚动。滚动过程中底部导航栏始终固定在屏幕底部可见，顶部导航栏始终固定在屏幕顶部可见，用户可随时切换 Tab 或返回。
- **场景 B**: 用户在钱包页进行充值操作，输入金额、选择快捷金额按钮、点击提交。所有表单元素在手机屏幕上排列合理，不出现溢出或重叠，虚拟键盘弹出时不遮挡关键操作按钮。
- **场景 C**: 用户在游戏大厅浏览游戏列表，游戏卡片网格在手机屏幕上排列整齐，无水平滚动条，卡片内容不截断。
- **场景 D**: 用户打开侧边菜单浏览功能入口，菜单在手机屏幕上完整展示，可流畅滚动，点击菜单项后正确跳转。
- **场景 E**: 用户在个人中心查看账户信息和设置项，各信息卡片和操作入口在手机屏幕上合理布局，文字不溢出。

## 背景

当前客户端 (frontend) 已有基础移动布局框架：
- BottomTabBar（64px 高度，5 个标签页）
- max-w-[430px] 固定最大宽度
- TopBar + 主内容区 + BottomTabBar 三段式布局

但在实际手机端使用中存在以下问题：
- 底部导航栏可能随页面内容一起滚动，导致用户需要回到页面顶部才能切换 Tab
- 部分页面在特定手机分辨率下出现内容溢出或布局错乱
- 长页面滚动体验不够流畅
- 部分表单和交互组件未针对触摸操作优化

本需求旨在系统性地解决这些手机端适配问题，提升移动端用户体验。

## 验收清单

- [ ] 底部导航栏 (BottomTabBar) 固定在视口底部，任何页面滚动时都不随内容移动
- [ ] 顶部导航栏 (TopBar) 固定在视口顶部，任何页面滚动时都不随内容移动
- [ ] 内容区域在 TopBar 和 BottomTabBar 之间独立滚动，不影响导航栏
- [ ] 首页长内容滚动流畅，滚动到底部后底部导航栏仍然可见
- [ ] 钱包页充值/提现表单在手机屏幕上布局合理，无溢出
- [ ] 游戏大厅游戏卡片网格在手机屏幕上排列正确，无水平滚动条
- [ ] 侧边菜单在手机端完整展示且可流畅滚动
- [ ] 个人中心页面在手机端布局合理
- [ ] 所有可点击元素的触控区域不小于 44x44px
- [ ] 在 iPhone SE (375px)、iPhone 14 (390px)、iPhone 14 Pro Max (430px) 三种宽度下均显示正常
- [ ] 无水平溢出导致的横向滚动条
- [ ] 虚拟键盘弹出时表单输入区域不被遮挡
