# FIX-2: 首页多余轮播图及支付方式布局不一致

> 日期: 2026-03-05 | 状态: 已修复 | 严重程度: P1

## 现象

1. 首页顶部 PromoBanner 下方多出一个大型 BannerCarousel 轮播图，设计稿中没有
2. 支付方式栏（UPI / Paytm / PhonePe / GPay / IMPS）用 SVG text 渲染，与设计稿中简单 span 文本样式不一致

## 根因分析

1. HomePage.tsx 渲染了 BannerCarousel 组件，该组件从 API 获取 banner 数据并展示为大型轮播，但设计稿中此位置不存在轮播图
2. PaymentMethodsBar.tsx 将每个支付方式用 SVG text 包裹，渲染的字体和间距与 HTML span 存在差异

## 修复方案

1. 从 HomePage.tsx 移除 BannerCarousel 渲染及其 import
2. 重写 PaymentMethodsBar.tsx，用 HTML span 替代 SVG text，颜色和字体样式与设计稿完全对齐

## 变更文件

| 文件 | 修改说明 |
|------|----------|
| `frontend/src/pages/home/HomePage.tsx` | 移除 BannerCarousel 组件渲染及 import |
| `frontend/src/pages/home/components/PaymentMethodsBar.tsx` | 用 HTML span 替换 SVG text，1:1 还原设计稿布局 |
