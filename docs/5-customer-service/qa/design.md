# QA 测试方案 -- 客服系统

> 需求: customer-service | 角色: qa

## 测试策略

| 层级 | 工具 | 范围 |
|------|------|------|
| API 测试 | Go httptest | /support/* 接口 |
| 前端测试 | 手动 | 对话窗口、链接跳转 |

## 测试用例

### 社交媒体链接

- API 返回 5 个平台链接
- 每个链接 URL 格式正确
- 前端点击正确跳转到外部平台

### 在线客服 (Mock)

- 侧边菜单入口打开对话窗口
- 发送消息后收到自动回复
- 关闭按钮正常工作

## 关键决策

- 客服系统测试以功能可用性为主
- Mock 对话只验证 UI 交互, 不验证回复内容质量
