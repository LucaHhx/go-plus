# UI 设计方案 -- 客服系统

> 需求: customer-service | 角色: ui

## 设计理念

客服系统设计轻量化。在线客服通过底部弹出面板实现，不中断当前浏览体验。社交媒体链接在首页页脚展示，方便用户选择偏好的沟通渠道。

## 设计规范

继承全局设计系统。

### 特有色值

| 用途 | 值 |
|------|-----|
| 客服消息气泡 | #2A2D2D (灰色) |
| 用户消息气泡 | #24EE89 (品牌色) |
| 用户消息文字 | #1A1D1D (深色) |
| Telegram | #0088CC |
| Facebook | #1877F2 |
| Instagram | 渐变 #833AB4 -> #FD1D1D -> #F77737 |
| WhatsApp | #25D366 |
| YouTube | #FF0000 |

## 页面设计

### 在线客服面板

- 触发: 侧边菜单 "Live Support" 点击
- 展现方式: 从底部滑上，高度 85vh (移动端近乎全屏)
- 背景: #232626
- 圆角: 顶部 16px

**面板布局:**

1. **顶部栏** (48px)
   - 左: "Live Support" 标题 (白色, bold)
   - 右: 关闭按钮 (X)
   - 底部 1px 分割线

2. **对话区** (可滚动)
   - 客服消息: 左对齐, #2A2D2D 背景气泡, 白色文字
   - 用户消息: 右对齐, #24EE89 背景气泡, 深色文字
   - 时间戳: 居中, 10px, #6B7070
   - 气泡最大宽度: 75%
   - 气泡圆角: 12px (对方角 4px)

3. **输入区** (56px)
   - 左: 输入框 (flex-1, 40px 高, 圆角 20px)
   - 右: 发送按钮 (品牌色圆形)

### 社交媒体链接 (也包含在客服 merge.html 中)

- 标题: "Community" (16px, bold, 白色, mb-4)
- 2 列网格 (gap-3):
  - Telegram (#0088CC 背景)
  - Facebook (#1877F2 背景, 文案 "facebook" 小写)
  - Instagram (渐变 from-#833AB4 via-#FD1D1D to-#F77737)
  - WhatsApp (#25D366 背景)
- YouTube (#FF0000 背景) -- **单独一行, w-fit** (非 2 列网格内)
- 按钮样式: flex, align-items center, gap-2, 圆角 8px, px-4 py-3
- 图标: 20px SVG, 白色 fill
- 文字: 14px, bold, 白色
- hover: opacity 0.9

## 组件规范

### 聊天气泡

| 类型 | 背景 | 文字 | 对齐 | 圆角 |
|------|------|------|------|------|
| 客服 | #2A2D2D | #FFFFFF | 左 | 4px 12px 12px 12px |
| 用户 | #24EE89 | #1A1D1D | 右 | 12px 4px 12px 12px |

### 客服头像

- 32px (w-8 h-8) 圆形
- 背景: brand/20 (品牌色 20% 透明度)
- 图标: 品牌色聊天气泡 SVG, 16px
- 位于每条客服消息左侧

### 打字指示器

- 同客服消息布局 (头像 + 气泡)
- 气泡内 3 个灰色圆点 (8px), bounce 动画
- 动画延迟: 0ms / 150ms / 300ms

### 发送按钮

- 圆形, 40px (w-10 h-10)
- 背景: #24EE89
- 图标: 白色发送箭头 SVG, 20px
- hover: #1DBF6E
- 禁用态: 输入为空时灰色背景

## 交互说明

- 面板从底部 300ms ease-out 滑入
- 背景遮罩 rgba(0,0,0,0.6)
- 点击遮罩或关闭按钮关闭面板
- 新消息自动滚动到底部
- 发送消息: Enter 或点击发送按钮

## 关键决策

1. 在线客服使用底部面板而非独立页面
2. 第一期对话功能是 Mock UI，后续接入第三方客服 SDK
3. 社交媒体链接复用首页页脚组件
4. Live Support 入口仅在侧边菜单，不在其他位置重复
