# QA 测试方案 -- 管理后台

> 需求: admin-panel | 角色: qa

## 测试策略

| 层级 | 工具 | 范围 |
|------|------|------|
| API 测试 | Go httptest | 所有 /api/admin/* 接口 |
| 权限测试 | Go httptest | Admin JWT 隔离验证 |
| 前端测试 | 手动 | Dashboard/CRUD 操作 |

## 测试用例

### 认证与权限

- 管理员登录成功 -> 返回 admin JWT
- 用客户端 JWT 访问 /api/admin/* -> 403
- 用管理 JWT 访问 /api/v1/* -> 正常 (不应阻止)

### Dashboard

- 统计数据返回正确结构
- 今日/近7日数据区间正确

### 用户管理

- 列表分页/搜索正确
- 禁用用户 -> 该用户无法登录
- 启用用户 -> 恢复登录

### 提现审核

- 待审核列表正确
- 审核通过 -> 用户余额正确 (解冻)
- 审核拒绝 -> 用户余额恢复

### Banner 管理

- CRUD 操作正确
- 新增 Banner 在前端首页显示

### 操作日志

- 每个管理操作产生日志记录

## 关键决策

- 管理后台重点测试权限隔离和提现审核流程
- 操作日志覆盖所有写操作
