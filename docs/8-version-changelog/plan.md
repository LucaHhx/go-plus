# 版本更新日志 (version-changelog)

> 为用户提供平台版本更新说明，运维人员通过后台使用 Markdown 格式编辑更新内容，前端在侧边菜单底部展示版本更新日志列表。

## 目标

1. 用户可以在侧边菜单底部查看平台的版本更新历史
2. 运维人员可以通过管理后台创建和编辑版本更新记录（Markdown 格式）
3. 版本更新记录按版本号和日期倒序排列，最新版本在前

## 范围

**包含:**

- 前端：侧边菜单底部的版本更新入口
- 前端：版本更新日志列表页（展示所有版本记录）
- 前端：版本更新详情展示（Markdown 内容渲染）
- 后台管理：版本更新记录的创建、编辑、删除
- 后台管理：Markdown 格式的内容编辑器
- 后台管理：版本记录的发布/下架状态管理
- 数据存储：版本更新记录存储在后端数据库

**不包含:**

- 版本更新的推送通知（不在本需求范围）
- 版本更新的强制升级提示（不在本需求范围）
- 客户端 App 版本检测与自动更新（不在本需求范围）
- 多语言版本更新内容（后续需求扩展）

## 核心用户场景

- **场景 A：用户查看版本更新日志** — 用户打开侧边菜单，在菜单底部区域看到"版本更新"入口（可显示当前版本号），点击后进入版本更新日志页面，看到按时间倒序排列的版本列表，每条包含版本号、更新日期和更新内容摘要，点击某条记录可展开查看完整的 Markdown 渲染内容。
- **场景 B：运维人员发布新版本更新记录** — 运维人员登录管理后台，进入"版本更新管理"页面，点击"新建版本记录"，填写版本号、更新日期，在 Markdown 编辑器中编写更新内容（支持标题、列表、加粗等格式），预览确认后发布，前端用户即可看到新的版本记录。
- **场景 C：运维人员编辑已有版本记录** — 运维人员在后台版本列表中找到需要修改的记录，点击编辑，修改 Markdown 内容后保存，前端实时生效。
- **场景 D：运维人员管理版本记录状态** — 运维人员可以将某条版本记录设为下架状态（前端不展示），或重新上架。

## 背景

参照 1goplus.com 网站，在侧边菜单底部有一个版本更新说明功能，展示平台的历次版本更新内容。我们需要实现类似的功能，让用户了解平台的最新改进和变化，同时让运维团队能够方便地管理更新内容。

参考网站 UI 特征：
- 入口位于侧边菜单/抽屉菜单的底部区域
- 更新日志按版本号和日期排列
- 每条记录包含：版本号、更新日期、更新内容描述
- 内容支持 Markdown 格式渲染（列表、标题等）

> [备注] 参考网站 1goplus.com 的具体 UI 样式供 UI 设计师参考，开发以最终设计稿为准。
